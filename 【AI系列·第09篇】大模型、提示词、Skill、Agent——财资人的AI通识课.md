# 大模型、提示词、Skill、Agent——财资人的 AI 通识课

文 / Parson资金研习社

---

前几篇讲了 DeepSeek、Claude、MiMo 怎么用、怎么接、怎么省钱。

但后台收到最多的留言不是"怎么装"，而是——

**"Parson，大模型到底是什么？提示词又是什么？Skill 和 Agent 有什么区别？"**

说白了，很多资金人对 AI 的认知还停留在"ChatGPT 能聊天"的阶段。术语一多就懵，工具一多就劝退。

这篇做一件事：**把 AI 的四个核心概念讲清楚，让零基础的资金人也能看懂。**

不写代码，不讲技术架构，只讲"你作为资金人，需要知道什么"。

---

## 一、大模型（LLM）——你的"数字大脑"

大模型，全称"大语言模型"（Large Language Model，LLM），是所有 AI 应用的底层引擎。

**打个比方：大模型就像一个刚从名校毕业的管培生。**

- 读过海量的书和资料（训练数据），知识面极广
- 能写、能算、能翻译、能分析——什么都能聊两句
- 但没有实际工作经验，不知道你公司的规矩和流程
- 你说什么它做什么，但你不说清楚，它就自己瞎猜

对资金人来说，大模型能做的事情已经很具体了：读一份 50 页的银行授信合同，提取关键条款（利率、期限、MAC 条款、交叉违约）；把 50 个子公司的 Excel 日报汇总成集团级报表；根据应收账款明细生成账龄分析；甚至帮你起草给银行的沟通邮件。这些以前要花半天做的事情，现在几分钟就能出初稿。

市面上的大模型有很多：OpenAI 的 GPT-4o、Anthropic 的 Claude、Google 的 Gemini、DeepSeek、小米的 MiMo、阿里的通义千问……它们的区别主要在三个方面：

| 维度 | 含义 | 资金人关心的点 |
|------|------|--------------|
| 智力水平 | 理解复杂指令、推理、计算的能力 | 能不能准确分析财务数据？ |
| 上下文窗口 | 一次能"记住"多少内容 | 能不能一次读完一份 50 页的授信合同？ |
| 价格 | 每百万 token 多少钱 | 每月 AI 费用控制在多少？ |

**资金人选模型的核心原则：够用就好，不必追最贵的。**

日常写报告、做分析、整理数据——DeepSeek V4 或 MiMo 就够了，便宜且能打。复杂的长文档分析、跨文件推理——Claude 更稳。没必要为了一篇日报用最贵的模型，就像没必要请麦肯锡帮你做一张 Excel 表。

## 二、提示词（Prompt）——你给 AI 的"工作指令"

提示词，就是你对 AI 说的话。

**打个比方：如果大模型是管培生，提示词就是你给它布置的任务单。**

任务单写得清楚，管培生干得漂亮；任务单写得模糊，管培生就给你交一份"正确的废话"。

来看一个资金场景的对比：

**模糊的提示词：**
> "帮我分析一下应收账款。"

AI 会给你一份泛泛而谈的"应收账款分析框架"——教科书式的，没有你公司的数据，没有具体的账龄分布，啥用没有。

**精准的提示词：**
> "根据附件的应收账款明细表，按账龄分级（0-30天、31-60天、61-90天、90天以上），按客户汇总应收账款余额和占比，标记逾期超过合同账期的客户，输出一张汇总表格。金额单位万元，保留两位小数。"

同一个 AI，输出质量天差地别。

**提示词的核心技巧就三条：**

1. **说清楚你要什么**——输出格式、数据范围、计算逻辑，越具体越好
2. **说清楚你不要什么**——"不要教科书式的框架""不要编造数据""缺失数据用N/A标注"
3. **给它一个角色**——"你是一个有10年经验的资金管理专家"，这句话能让 AI 的输出质量提升一个档次

来看一个"给角色"前后的对比：

> 不给角色："帮我写一封邮件，跟银行谈降低贷款利率。"
> AI 输出：一封措辞礼貌但毫无说服力的通用邮件。

> 给角色："你是一个有10年银企关系管理经验的资金总监。请帮我写一封邮件给XX银行的客户经理，核心诉求是将流贷利率从LPR+80bp降到LPR+50bp。谈判筹码：集团在该行有2亿存款沉淀、全年结算量超过50亿、且有多家银行竞争。语气要专业但有底气。"
> AI 输出：一封有数据支撑、有谈判策略、有底线的专业邮件。

差距一目了然。提示词不是"问问题"，是"写任务brief"。

但提示词有一个致命问题：**一次性，用完就扔。**

你今天花 20 分钟打磨了一个完美的资金日报提示词，明天换个场景又得从头写。这就是为什么需要 Skill。

## 三、Skill——你的"标准作业手册"

Skill 不是提示词。提示词是一次性的口头交代，Skill 是写进操作手册的标准流程。

**打个比方：提示词是口头跟新员工说"帮我整理一下这个表格"，Skill 是你给他发了一份《资金日报标准化操作手册》，里面写清楚了每一步怎么做、格式是什么、哪些错误必须避免。**

在 Claude Code 里，Skill 就是一个 Markdown 文件，保存在项目的 `Skills/` 目录下。你输入一个命令或者说一句话，AI 自动加载这个文件，按照里面的流程执行。

一个好的 Skill 包含五个模块：

| 模块 | 作用 | 资金场景示例 |
|------|------|------------|
| 元数据 | 告诉系统这个 Skill 是什么 | name: daily-report |
| 触发词 | 什么时候该用这个 Skill | "生成资金日报""做日报""今天的头寸" |
| 工作流程 | 一步一步告诉 AI 做什么 | 收集数据→清洗→生成报表→标注异常 |
| 约束条件 | 哪些事不能做 | 金额单位万元、币种用 ISO 编码、不编造数据 |
| 输出模板 | 最终输出长什么样 | 头寸总览表+子公司明细+大额收支+异常标注 |

**Skill 的核心价值是三个字：可复用。**

你花 3 天打磨的资金日报 Skill，团队里每个人都能一键执行。你请假了，同事输入 `/资金日报`，AI 输出的格式跟你做的一模一样。

对资金人来说，Skill 库就是你的**专业能力的数字化沉淀**。你积累的日报流程、授信分析框架、银行评估矩阵，都可以封装成 Skill。

打个更直白的比方：提示词是你口头上跟实习生说"帮我做个表"，结果每次做出来的都不一样。Skill 是你写了一份带截图的操作手册——谁来做，结果都一样。资金部最怕的就是"关键人请假就瘫痪"，Skill 能解决这个问题。

## 四、Agent——你的"数字员工"

Agent 是 AI 的终极形态。

**打个比方：大模型是大脑，提示词是临时指令，Skill 是操作手册，Agent 是一个能自己看手册、自己干活、自己检查结果的员工。**

Agent 和普通 AI 对话最大的区别在于三个字：**自主性。**

| 维度 | 普通 AI 对话 | Agent |
|------|------------|-------|
| 工作方式 | 你问一句，它答一句 | 你给一个目标，它自己拆解步骤、逐步执行 |
| 工具使用 | 只能聊天 | 能读文件、写文件、跑命令、调 API |
| 记忆 | 每次对话从零开始 | 能记住上下文、记住你的偏好 |
| 纠错 | 你发现错误，手动纠正 | 它自己检查结果，发现问题自动修正 |

举个实际例子：

你对普通 AI 说："帮我把这 50 个子公司的资金日报汇总成集团报表。"它会告诉你"好的，请上传文件"——然后你得手动上传 50 个文件，一个一个来。

你对 Agent 说同样的话，它会：
1. 自己找到文件夹里的 50 个 Excel
2. 逐个读取，识别数据结构
3. 清洗数据（统一币种、统一单位、标记异常）
4. 汇总生成集团级报表
5. 检查有没有遗漏或错误
6. 把成品保存到指定位置

**整个过程你只需要说一句话。**

在前面几篇提到的 Claude Code，本质上就是一个 Agent——它住在你的终端里，能读你的文件、跑命令、写代码、管项目。WorkBuddy 也是 Agent——它住在你的桌面上，能替你操作文件和应用。

Agent 最厉害的地方不是"聪明"，而是**能组合使用多种工具**。一个 Agent 可以同时读 Excel、查数据库、调银行 API、生成图表、写报告——就像一个员工同时会用 SAP、网银、Excel 和 Word。以前你需要在五个系统之间来回切换，现在 Agent 替你串起来了。

当然，Agent 不是万能的。它执行的每一步都是基于你给的指令和它学到的规则。如果指令有歧义，它可能走偏；如果数据本身有错，它不会自动发现。所以资金人用 Agent 的正确姿势是：**让它干活，但你来验收。**

## 五、四个概念的关系——一张图说清楚

把四个概念串起来，它们的关系是这样的：

```
大模型（LLM）= 大脑（有知识，没经验）
    ↓ 给它指令
提示词（Prompt）= 临时任务单（一次性）
    ↓ 固化成流程
Skill = 标准操作手册（可复用）
    ↓ 让它自主执行
Agent = 数字员工（能自己干活）
```

越往下，AI 的自主性越强，你需要操心的事情越少。

**资金人的进阶路径也是这四步：**

1. **会用大模型**：知道 ChatGPT/Claude/DeepSeek 能干什么，会问问题
2. **会写提示词**：能把你的需求描述清楚，让 AI 输出高质量结果
3. **会做 Skill**：把你的专业流程封装成可复用的 AI 指令
4. **会用 Agent**：让 AI 自主完成复杂的多步骤任务

大多数资金人目前在第 1 步，少数在第 2 步。能走到第 3、4 步的，在团队里基本就是"AI 担当"了。

## 六、资金人最常问的五个问题

**Q1：不懂代码的财务人，能用这些吗？**

能。大模型和提示词完全不需要代码——你会打字就行。Skill 和 Agent 需要一点学习成本，但比学 Excel 公式简单。前面几篇写的安装教程，照着做就行。

**Q2：AI 会不会编造数据？**

会。这是大模型的"幻觉"问题——它会一本正经地编造不存在的数据和案例。应对方法有两个：一是提示词里明确写"不得编造数据，不确定的用[需人工补充]标注"；二是关键数据必须人工复核。AI 是助手，不是决策者。

**Q3：用 AI 处理公司财务数据，安全吗？**

取决于你用什么工具。直接把数据粘到 ChatGPT 网页版——数据会上传到 OpenAI 的服务器，有泄露风险。用 Claude Code + 本地模型（如 DeepSeek）——数据不出你的电脑，安全性高很多。具体怎么选，可以看前面几篇的方案对比。

**Q4：提示词写多长合适？**

不是越长越好。好的提示词像一份清晰的任务单：目标明确、约束清楚、格式具体。通常 200-500 字就够了。超过 1000 字的提示词，往往是因为把"背景知识"和"具体指令"混在了一起——背景知识应该放在 Skill 里，不是每次都要重复。

**Q5：资金人应该从哪里开始？**

从你最重复、最耗时的那件事开始。每天花 30 分钟做的资金日报？每周花 2 小时做的授信台账整理？每月花 1 天做的银行对账？挑一件，用 AI 做一遍。第一次可能花的时间比手工还长，但第二次、第三次就会越来越快。等你把这件事封装成 Skill，就再也不用自己动手了。

具体建议的起步顺序：

1. **第一周**：注册一个AI账号（豆包，千问，kimi都可以），试着让它帮你写一封银行沟通邮件、分析一份简单的 Excel 数据
2. **第二周**：试着用更精准的提示词——把你的需求写成"目标+约束+格式"的三段式
3. **第三周**：把你最熟练的那件重复性工作，写成一份 Skill 的初稿（哪怕只有 200 字）
4. **第一个月后**：试着用 Claude Code 或 WorkBuddy 这类 Agent 工具，让它替你执行你写好的 Skill

不用急，一步步来。AI 是工具，不是魔法——用得越多，手感越好。最怕的不是"用得不好"，而是"一直不用"。

## 写在最后

AI 不是什么神秘的黑科技。

大模型是有知识没经验的管培生，提示词是你布置的任务单，Skill 是你写的操作手册，Agent 是能自己干活的数字员工。

**理解了这四层关系，你就不会再被各种术语吓到。**

资金人不需要成为 AI 专家，但需要成为**会用 AI 的资金专家**。

从今天开始，挑一件你最烦的重复性工作，试着交给 AI 做一遍。

你会发现，它比你想象的能干——也比你想象的笨。但只要你会"教"，它就会越来越像一个靠谱的助手。

2026 年，不会用 AI 的资金人不会被淘汰，但会用 AI 的资金人会把效率拉开一个量级。这个差距，早补比晚补容易。
