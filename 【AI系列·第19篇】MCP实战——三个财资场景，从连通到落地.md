## 实战一：用 WPS KnowledgeBase MCP 管理知识库

### 场景

你是集团资金部的人，WPS 知识库里有 200 多份文档——资金日报模板、授信台账、合同范本、分析报告。每次要找一个模板，都要打开浏览器、登录 WPS、一层层点进去。

现在，用 Claude Code + WPS KnowledgeBase MCP，直接在终端里操作。

### 第一步：安装 MCP 服务器

WPS KnowledgeBase Skill 自带 MCP 安装脚本。如果你已经装了这个 Skill，运行：

```bash
cd ~/.claude/skills
bash scripts/setup.sh
```

脚本会自动完成三件事：
1. 安装 `mcporter`（MCP 服务管理工具）
2. 获取 WPS 认证 Token
3. 注册 MCP 服务器

安装完成后，运行验证命令：

```bash
mcporter list kwiki
```

返回工具列表，说明安装成功。

### 第二步：在 Claude Code 里调用

打开 Claude Code，直接说人话：

```
> 帮我列出所有知识库
```

Claude Code 会调用 `kwiki.list_knowledge_view` 工具，返回你所有的知识库列表：

```
知识库列表：
1. 集团资金管理库（kuid: abc123）
2. 授信合同模板库（kuid: def456）
3. 子公司日报模板库（kuid: ghi789）
...
```

继续说：

```
> 在"子公司日报模板库"里找一下"资金日报模板.xlsx"
```

Claude Code 调用 `kwiki.list_file` 遍历文件列表，找到目标文件，返回文件信息和直达链接。

### 第三步：批量操作

这才是 MCP 的价值——批量操作。

```
> 把"子公司日报模板库"里所有 2025 年的模板，全部移动到"历史模板"文件夹
```

Claude Code 会：
1. 调用 `kwiki.list_file` 列出所有文件
2. 按名称筛选出 2025 年的模板
3. 调用 `kwiki.create_file` 创建"历史模板"文件夹（如果不存在）
4. 逐个调用 `kwiki.move_file` 移动文件
5. 返回操作结果（成功 X 个，失败 Y 个）

**你只需要说一句话，AI 帮你完成 20 次点击。**

### 避坑提示

1. **写操作需要 CSRF Token。** WPS KnowledgeBase 的所有写操作（创建、修改、删除、移动）都要求携带 CSRF Token。MCP 服务器会自动处理，但如果你自己写脚本调用，需要手动生成一个 32 位十六进制字符串。

2. **Token 会过期。** WPS 的认证 Token 有有效期，过期后需要重新获取。Skill 的 `setup.sh` 脚本会启动一个保活进程（每 5 分钟续期），但如果电脑休眠了可能断开。遇到认证失败，重新运行 `setup.sh` 即可。

3. **文件链接格式。** WPS 返回的链接是相对路径，拼接规则：`https://www.kdocs.cn` + 返回的 `url` 字段。如果返回的是 `kuid`，格式为 `https://www.kdocs.cn/wiki/l/${kuid}`。

## 实战二：用浏览器 MCP 抓取汇率数据

### 场景

你每天要从央行网站或汇率查询网站上抄当日汇率，更新到公司的汇率表里。这件事不难，但每天重复做很烦。

用 Playwright MCP，让 AI 自动打开浏览器、抓取数据、填入表格。

### 第一步：安装 Playwright MCP

在项目根目录的 `.mcp.json` 里添加：

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-playwright"]
    }
  }
}
```

首次使用时，`npx` 会自动下载安装。

### 第二步：抓取汇率

打开 Claude Code：

```
> 帮我打开中国银行外汇牌价页面，抓取今天美元、欧元、日元的现汇买入价
```

Claude Code 会：
1. 调用 Playwright MCP 的 `browser_navigate` 工具，打开中国银行外汇牌价页面
2. 调用 `browser_snapshot` 或 `browser_screenshot` 获取页面内容
3. 解析页面数据，提取美元、欧元、日元的现汇买入价
4. 返回结果

```
今日外汇牌价（现汇买入价）：
- 美元 USD：718.35
- 欧元 EUR：782.46
- 日元 JPY：4.8156
```

### 第三步：自动写入汇率表

如果同时配置了 Filesystem MCP，可以一步到位：

```
> 把今天抓到的汇率写入"汇率表.xlsx"的新一行，日期用今天的
```

Claude Code 调用 Filesystem MCP 读取 Excel 文件，追加一行新数据，保存。

### 进阶用法

```
> 每天早上 9 点自动抓取汇率，如果美元汇率超过 7.20，发一条预警到飞书群
```

这句话需要组合三个 MCP：
- Playwright MCP 抓汇率
- Filesystem MCP 读写汇率表
- 飞书 MCP 发预警消息

再加上一个定时触发（可以用系统的 cron 或 Claude Code 的 hooks 功能），就构成了一个完整的自动化流程。

### 避坑提示

1. **网站结构会变。** 银行网站改版后，页面元素的位置和名称可能变化，导致抓取失败。建议每季度检查一次抓取脚本是否正常。

2. **反爬虫机制。** 部分网站有反爬虫措施，频繁抓取可能被封 IP。Playwright MCP 模拟的是真实浏览器行为，比纯 HTTP 请求更不容易被拦截，但仍需注意频率。

3. **数据校验。** 抓到的汇率一定要跟人工抽查对比。AI 抓取的数据可能有误（页面加载不完整、数据格式异常等），前几次使用务必人工复核。

## 实战三：用数据库 MCP 查询资金流水

### 场景

你想查子公司 A 上个月的资金流入流出情况，传统方式是打开 ERP 系统、设置筛选条件、导出 Excel、再用数据透视表汇总。

用数据库 MCP，直接用自然语言查。

### 第一步：安装数据库 MCP

以 PostgreSQL 为例，在 `.mcp.json` 里添加：

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://user:password@host:5432/dbname"]
    }
  }
}
```

**安全警告：** 数据库连接字符串包含账号密码，**绝对不要**把 `.mcp.json` 提交到公开的 Git 仓库。建议用环境变量：

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "${DATABASE_URL}"]
    }
  }
}
```

### 第二步：用自然语言查询

打开 Claude Code：

```
> 查一下子公司 A（编号 SUB001）2026 年 5 月的资金流入流出汇总，按交易类型分组
```

Claude Code 会：
1. 调用数据库 MCP 的 `query` 工具
2. 自动将自然语言翻译成 SQL：

```sql
SELECT 
  transaction_type,
  SUM(amount) as total_amount,
  COUNT(*) as transaction_count
FROM fund_transactions
WHERE subsidiary_id = 'SUB001'
  AND transaction_date >= '2026-05-01'
  AND transaction_date <= '2026-05-31'
GROUP BY transaction_type
ORDER BY total_amount DESC;
```

3. 返回查询结果

```
子公司 A 2026 年 5 月资金流水汇总：
| 交易类型 | 金额（万元） | 笔数 |
|---------|------------|------|
| 销售回款 | 8,542.30 | 156 |
| 采购付款 | -6,213.80 | 89 |
| 工资发放 | -1,245.00 | 1 |
| 银行利息 | 12.35 | 3 |
| 合计 | 1,095.85 | 249 |
```

### 进阶用法

```
> 查一下所有子公司 5 月的资金净流入，找出净流出超过 500 万的子公司
```

```
> 对比子公司 A 和子公司 B 的资金流水，看看哪个回款效率更高
```

```
> 把查询结果生成一份 Excel，发到我的 WPS 知识库里
```

最后这句话组合了三个 MCP——数据库查询 + 文件生成 + WPS 上传。

### 避坑提示

1. **只读原则。** 建议给 MCP 使用的数据库账号设置**只读权限**。AI 不应该有写入生产数据库的能力——查询可以，修改不行。

2. **SQL 注入风险。** MCP 服务器会处理参数化查询，但如果你让 AI 直接拼接 SQL，仍有注入风险。建议在数据库侧配置查询白名单，只允许 SELECT 操作。

3. **大数据量查询。** 如果表有几百万行记录，AI 生成的 SQL 可能没有加 LIMIT，导致查询超时。建议在 MCP 配置里设置默认的查询超时时间和结果行数限制。

4. **数据敏感性。** 资金流水是高度敏感数据。确保 Claude Code 的对话记录不会泄露——参考第 13 篇的数据安全指南，用本地模型或私有化部署方案处理敏感数据。

## 组合使用：MCP + Skill 的完整工作流

三个场景分别用了不同的 MCP 服务器，但真正的威力在于**组合使用**。

举一个完整的例子：**每日资金日报自动化。**

```
早上 8:30，你打开 Claude Code，说："开始今天的资金日报。"

Claude Code 按照 Skill 里定义的流程，依次执行：

1. 【数据库 MCP】查询所有子公司昨日的银行余额和交易流水
2. 【Filesystem MCP】读取集团日报模板 Excel
3. 【AI 模型】清洗数据、统一格式、计算头寸缺口
4. 【Filesystem MCP】写入日报模板，生成今日日报
5. 【WPS MCP】上传到 WPS 知识库"每日日报"文件夹
6. 【飞书 MCP】在"集团资金日报群"发一条消息："今日日报已更新，链接：xxx"
7. 【飞书 MCP】如果有子公司余额超过授信额度，额外发一条预警

全程：你只说了一句话。耗时：约 3 分钟。
```

这就是 MCP + Skill 的完整工作流——Skill 定义"做什么"，MCP 提供"怎么连"，AI 负责"怎么做"。

## 五、常见问题排查

### 问题一：MCP 服务器启动失败

**症状：** Claude Code 提示"无法连接到 MCP 服务器"

**排查步骤：**

1. 检查 `.mcp.json` 里的 `command` 和 `args` 是否正确
2. 手动在终端运行 `npx -y @xxx/mcp-server`，看有没有报错
3. 如果是自定义服务器，检查端口是否被占用
4. 运行 `claude --mcp-debug` 查看详细日志

### 问题二：工具调用成功但返回空结果

**症状：** AI 说"已调用工具"，但返回数据为空

**可能原因：**

1. 数据库查询条件太严格，没有匹配的记录
2. WPS 知识库的文件夹 ID 不对
3. 网页抓取时页面还没加载完就获取了内容

**解决方法：** 让 AI 先用最简单的查询验证连通性（比如 `SELECT 1` 或列出所有知识库），确认连接正常后再加筛选条件。

### 问题三：认证过期

**症状：** 之前能用，突然报认证错误

**解决方法：** 重新运行 MCP 服务器的认证脚本。WPS 知识库是 `bash scripts/get-token.sh`，数据库是检查连接字符串是否变更。

## 六、MCP 的安全边界

第 13 篇讲过数据安全，这里再强调几个 MCP 相关的安全要点：

| 风险 | 应对措施 |
|------|---------|
| 数据库连接字符串泄露 | 用环境变量，不写在配置文件里 |
| AI 误操作写入数据库 | 数据库账号设只读权限 |
| 对话记录包含敏感数据 | 用本地模型或私有化部署 |
| MCP 服务器被篡改 | 只用官方或可信来源的 MCP 服务器 |
| 资金数据通过 API 传输 | 确认 MCP 服务器走 HTTPS，不走明文 HTTP |

**核心原则：MCP 扩展了 AI 的能力边界，但也扩大了风险边界。** 连接的系统越多，需要管控的点越多。建议循序渐进——先连一个系统跑通，确认安全可控后再扩展。

## 七、财资人的 MCP 建设路径

不要一口气装太多 MCP 服务器。建议分三个阶段：

**第一阶段：只读查询（1-2 周）**

装数据库 MCP，用只读账号，让 AI 帮你查数据、做汇总。这个阶段没有写操作风险，放心用。

**第二阶段：文档管理（2-4 周）**

装 WPS KnowledgeBase MCP 或 Filesystem MCP，让 AI 帮你管理文档、整理模板、批量操作。这个阶段有写操作，但写的是文档不是数据库，风险可控。

**第三阶段：系统联动（1-2 个月）**

装飞书/企业微信 MCP，让 AI 发消息、推预警。再装浏览器 MCP，自动抓取外部数据。这个阶段开始涉及多系统联动，需要更多测试和安全审查。
