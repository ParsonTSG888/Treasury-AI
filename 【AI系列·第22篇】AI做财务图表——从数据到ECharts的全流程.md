前 21 篇讲了 AI 怎么查数据、怎么写报告、怎么审合同。但有一个场景一直没碰：**可视化。**

资金人做汇报，最头疼的不是"数据对不对"，而是"图表好不好看"。

CFO 看一份 50 页的文字报告，3 分钟就走神了。但如果你给他一张资金头寸趋势图、一张子公司余额分布饼图、一个实时滚动的仪表盘——他能盯着看 10 分钟，还能在董事会上直接用。

**数据可视化不是"锦上添花"，是"让数据被看见"。**

问题是：大多数资金人不会写代码，更不会用 D3.js 或 ECharts。

好消息：AI 会。

## 一、为什么选 ECharts

市面上的可视化工具一堆——Excel 图表、Tableau、Power BI、Python Matplotlib、D3.js、ECharts……为什么这一篇专门讲 ECharts？

三个原因：

**1. 零安装、零成本。** ECharts 是百度开源的 JavaScript 图表库，纯前端，不需要安装任何软件。一个 HTML 文件就能跑。Tableau 要付费，Power BI 要装桌面端，Python 要配环境——ECharts 什么都不需要。

**2. 效果碾压 Excel。** Excel 图表能用，但丑。ECharts 的默认样式就很好看，而且支持交互（鼠标悬停显示数值、点击切换维度、缩放时间轴）。汇报时用 ECharts 图表，视觉效果直接拉满。

**3. AI 生成 ECharts 代码非常准确。** ECharts 的配置格式是结构化的 JSON，大模型对 JSON 的理解非常精准。你让 AI 生成一个 ECharts 折线图，90% 的情况下一次就能跑对。D3.js 的代码逻辑更复杂，AI 生成的准确率反而低。

**一句话：ECharts 是资金人做数据可视化的最佳起点。**

## 二、第一个图表：5 分钟生成

### 第一步：准备数据

假设你有一份子公司资金余额数据：

```
子公司A: 1,258 万元
子公司B: 3,421 万元
子公司C: 892 万元
子公司D: 2,156 万元
子公司E: 1,673 万元
```

### 第二步：告诉 AI 你要什么

打开 Claude Code：

```
> 帮我用 ECharts 生成一个柱状图，展示各子公司的资金余额。
> 数据：子公司A 1258万，子公司B 3421万，子公司C 892万，子公司D 2156万，子公司E 1673万。
> 输出一个完整的 HTML 文件。
```

### 第三步：AI 生成代码

Claude Code 会生成一个完整的 HTML 文件：

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>子公司资金余额</title>
  <script src="https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js"></script>
</head>
<body>
  <div id="chart" style="width: 800px; height: 500px;"></div>
  <script>
    var chart = echarts.init(document.getElementById('chart'));
    var option = {
      title: { text: '子公司资金余额（万元）' },
      tooltip: { trigger: 'axis' },
      xAxis: { 
        type: 'category',
        data: ['子公司A', '子公司B', '子公司C', '子公司D', '子公司E']
      },
      yAxis: { type: 'value', name: '万元' },
      series: [{
        type: 'bar',
        data: [1258, 3421, 892, 2156, 1673],
        itemStyle: {
          color: function(params) {
            var colors = ['#5470c6', '#91cc75', '#fac858', '#ee6666', '#73c0de'];
            return colors[params.dataIndex];
          }
        }
      }]
    };
    chart.setOption(option);
  </script>
</body>
</html>
```

### 第四步：查看效果

双击生成的 HTML 文件，浏览器自动打开，一个漂亮的柱状图就出来了。

**鼠标悬停在柱子上，会显示具体数值。** 这是 ECharts 的默认交互行为，不需要额外配置。

## 三、五种财务常用图表

### 图表一：折线图——资金头寸趋势

**场景：** 展示过去 30 天的资金头寸变化趋势。

```
> 生成一个折线图，展示过去 30 天的资金头寸变化。
> 数据我用一个 CSV 文件：date,balance（你替换成真实数据）。
```

AI 会读取 CSV 文件，生成带时间轴的折线图。支持缩放——鼠标滚轮可以放大某一天的数据。

**进阶：** 加一条"预警线"，当余额低于某个阈值时，线条变红。

```javascript
markLine: {
  data: [{ yAxis: 1000, name: '预警线' }],
  lineStyle: { color: 'red', type: 'dashed' }
}
```

### 图表二：饼图——资金分布

**场景：** 展示各子公司的资金占比。

```
> 生成一个饼图，展示 5 个子公司的资金余额占比。
```

**进阶：** 用环形图（donut chart）替代饼图——视觉效果更现代。在 ECharts 里，只需要把 `type: 'pie'` 改成 `radius: ['40%', '70%']` 就行。

### 图表三：柱状图+折线图组合——收入与增长率

**场景：** 左轴是各子公司的收入（柱状图），右轴是同比增长率（折线图）。

```
> 生成一个柱状图+折线图的组合图。
> 柱状图展示各子公司的收入，折线图展示同比增长率。
> 左轴是收入（万元），右轴是增长率（%）。
```

这种组合图在汇报中非常常见——一张图同时展示绝对值和相对变化。

### 图表四：仪表盘——资金充足率

**场景：** 用一个仪表盘展示当前资金充足率，指针指向当前值，颜色分绿/黄/红三个区间。

```
> 生成一个仪表盘，展示资金充足率。
> 当前值：78%。绿色区间 70-100%，黄色区间 50-70%，红色区间 0-50%。
```

这种图表在资金监控看板上特别醒目——CFO 一眼就能看到资金状态。

### 图表五：热力图——子公司资金变动矩阵

**场景：** 横轴是时间（月份），纵轴是子公司名称，颜色深浅表示资金变动幅度。

```
> 生成一个热力图，展示各子公司每月的资金变动情况。
> 颜色深浅表示变动幅度，深红表示大额流出，深绿表示大额流入。
```

热力图适合展示"谁在什么时候动了多少钱"——比表格直观 10 倍。

## 四、从 Excel 到 ECharts：完整工作流

前面的例子都是手动输入数据。实际工作中，数据通常在 Excel 里。

### 第一步：AI 读取 Excel

```
> 读取"子公司余额.xlsx"，用 ECharts 生成柱状图。
```

Claude Code 会用 Python 读取 Excel 文件，提取数据，然后生成 ECharts 配置。

### 第二步：自动生图

```
> 读取"子公司余额.xlsx"，生成以下图表：
> 1. 各子公司余额柱状图
> 2. 资金分布饼图
> 3. 过去 30 天趋势折线图
> 全部输出到一个 HTML 文件里，左右分栏显示。
```

AI 会生成一个包含多个图表的 HTML 页面，自动布局，打开就能看。

### 第三步：嵌入知识库

生成的 HTML 文件可以：
- 直接用浏览器打开查看
- 上传到 WPS 知识库，分享给团队
- 嵌入到内部系统的 iframe 里

```
> 把生成的图表上传到 WPS 知识库的"资金监控"文件夹。
```

## 五、美化技巧

### 技巧一：统一配色

告诉 AI 用企业品牌色：

```
> 图表的主色调用 #1a5276（深蓝），辅助色用 #2ecc71（绿）和 #e74c3c（红）。
```

### 技巧二：加数据标签

```
> 在每个柱子上方显示具体数值，格式：xxx 万元。
```

### 技巧三：响应式布局

```
> 图表宽度自适应浏览器窗口，最小宽度 600px。
```

### 技巧四：暗色主题

```
> 用 ECharts 的 dark 主题，背景深色，适合投影仪演示。
```

**一条提示词就能切换风格，AI 会在 5 秒内重新生成代码。**

## 六、避坑指南

### 坑一：数据格式不一致

Excel 里的数据可能是"1,258.47 万元"（带逗号和单位），AI 直接用会报错。

**解决：** 告诉 AI 先清洗数据——去掉逗号和单位，转成纯数字。

### 坑二：中文乱码

ECharts 默认支持 UTF-8。如果你的 HTML 文件在浏览器里显示乱码，检查 `<head>` 里有没有 `<meta charset="UTF-8">`。

### 坑三：CDN 加载失败

ECharts 的 JS 文件从 CDN 加载（`cdn.jsdelivr.net`）。如果你的电脑无法访问外网，图表会显示空白。

**解决：** 让 AI 把 ECharts 的 JS 文件下载到本地，改成相对路径引用。

### 坑四：数据量太大

如果数据有几千个点（比如每天的分钟级资金流水），折线图会卡顿。

**解决：** 让 AI 做数据聚合——按小时或按天汇总，减少数据点数量。
